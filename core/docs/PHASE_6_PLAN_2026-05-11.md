### Part A ‚Äî Finish Open Gaps (Week 1)

**Goal:** Make the inbound flow fully enforced before adding new modules.

#### Task A1 ‚Äî GRN Service Guards

- File: `core/src/main/java/com/batchsphere/core/transactions/grn/service/GrnServiceImpl.java`
- Add in `createGrn()`:
  - Load `Material` ‚Üí assert `status == ACTIVE`
  - Load `Supplier` ‚Üí assert `qualificationStatus != SUSPENDED && != DISQUALIFIED`
  - Load `VendorBusinessUnit` ‚Üí assert `qualificationStatus != DISQUALIFIED`
  - Query `VendorMaterialApproval` for (supplierId/vendorBusinessUnitId, materialId) ‚Üí assert at least one APPROVED record exists
- Add in `receiveGrn()`:
  - Load material's linked spec ‚Üí assert spec `status == APPROVED`
- Throw `BadRequestException` with descriptive message for each violation.
- Roles: all GRN-creating roles affected (WAREHOUSE_OP, SUPER_ADMIN).

#### Task A2 ‚Äî QcWorksheet Mandatory Completion Gate

- File: `QcWorksheetServiceImpl.java`
- Before `submitDisposition()` is allowed, call `isWorksheetComplete()`:
  - If any mandatory `SpecParameter` has no corresponding `QcTestResult` with `status = COMPLETED`, throw exception.
- Add this guard in `QcDispositionServiceImpl` / `SamplingServiceImpl.qcDecision()`.

#### Task A3 ‚Äî CoA Review Fields on GRN

Status: implemented.

- Migration: `V67__add_grn_coa_review_fields.sql`
- Add `CoaReviewStatus` enum to `com.batchsphere.core.transactions.grn.entity`
- Add fields to `Grn.java` entity and `GrnResponse` DTO
- Add `POST /api/grns/{id}/coa-review` endpoint ‚Äî accepts `CoaReviewRequest` (status, remarks)
- Role: QC_ANALYST or QC_MANAGER can submit CoA review

#### Task A4 ‚Äî Inventory Expiry Auto-Block Scheduler

- New file: `InventoryExpiryScheduler.java` in `transactions/inventory/service/`
- Runs daily at 02:00
- Queries `Inventory` where `expiryDate < now()` and `status NOT IN (REJECTED, BLOCKED)`
- Calls `InventoryService.updateStatus(id, BLOCKED)` for each
- Logs an `AuditEvent` for each auto-block with `actor = SYSTEM`
- Similar for `retestDueDate` ‚Äî moves to a logical hold state (log only, do not auto-block without QC input ‚Äî flag in UI instead)

- QMS for in-process check deviations (Phase 6 Part B)
- HRMS for dispensing traceability (Phase 6 Part C)
- LIMS for finished product testing

Defer until Phase 7 or later after all above are stable.

### 7.6 Recall Traceability

Forward-trace query: given a GRN batch number, find all inventory lots, sampling requests, manufacturing orders, and finished product batches that used this material.

No new entity needed ‚Äî build as a reporting query endpoint `GET /api/recall/trace?batchId={id}` joining existing tables.

### 7.7 API Documentation (Swagger/OpenAPI)

Add `springdoc-openapi-starter-webmvc-ui` dependency. Annotate controllers with `@Tag`, `@Operation`, `@ApiResponse`. Available at `/swagger-ui.html`.

Low effort, high value for any new developer or tester joining the project.

```
CycleCountRecord
  warehouseId, conductedBy, conductedAt

```

Reconciliation report + adjustment transaction creation on variance acceptance.

### 7.9 ChangeRequest (QMS extension)

Gate edits to Spec, Material, and Vendor through a formal change control workflow before these modules are submission-ready.

```
ChangeRequest
```
